补充：Django中使用模型查表，需要进行优化：

  • 查询后面加上only() / defer() - 只查/不查
  • 查询关联关系时，如果不加上限定语句，会出现1+N现象，导致查询效率十分低下
    1. 一对多：select_related(‘xxx’)
    2. 多对多：prefetch_related(‘xxx’)

说明：

  1. @api_view(['GET', 'POST'])指定方式；

  2. @cache_page(timeout=None, cache='api')指定使用缓存和超时时间；

  3. @throttle_classes(())设置当前接口不被限流。

基于类的视图 - CBV - Class-Based View
  1. 继承APIView及其子类(ListAPIView / CreateAPIView / …)
  1. 继承ModelViewSet / ReadOnlyModelViewSet - 视图函数

实例

说明：

  1. @method_decorator(cache_page(timeout=30, cache='api'), name='get')将类下面的指定请求方法设置缓存;

  2. 继承ListAPIView, CreateAPIView, UpdateAPIView, DestroyAPIView的CBV，接口对应拥有查看，创建，更新，删除方法；

  3. 继承ModelViewSet的CBV，接口拥有所有方法。其它类自定查找。

  4. 继承CacheResponseMixin的CBV，指定使用缓存，需要安装drf-extensions库支持，并进行下面配置；

  5. queryset，serializer_class分别对应重写的序列化对象和类。
