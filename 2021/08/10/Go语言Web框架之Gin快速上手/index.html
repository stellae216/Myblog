Go语言Web框架之Gin快速上手

本文最后更新于：6 个月前

原生Web服务

Go语言里面提供了一个完善的net/http包，通过http包可以很方便的搭建起来一个可以运行的Web服务。同时使用这个包能很简单地对Web的路由，静态文件，模版，cookie等数据进行设置和操作。

Web框架Gin

Gin 是一个用 Go (Golang) 编写的 HTTP web 框架。 它是一个类似于 martini 但拥有更好性能的 API 框架, 优于 httprouter，速度提高了近 40 倍。 如果你是性能和高效的追求者, 你会爱上 Gin

  • 模型参数建议使用指针类型，在不为required时，对应零值为nil，insert db时能够方便转换成null进行存储。参考这篇文章gin框架参数零值json绑定的问题

特殊的日期解析校验

Go 自身的 time.Time 类型默认解析的日期格式是 RFC3339 标准，也就是 2006-01-02T15:04:05Z 的格式。

但是如果我们想要在 Gin 的 shouldBindJSON 方法中，传入 YYYY-MM-DD hh:mm:ss 格式的日期格式作为 time.Time 类型的值，就会引发类似于 parsing time xx as xx: cannot parse xx as xx 的报错信息，这是因为 time.Time 类型默认支持的日期格式与我们传入的格式不同，导致解析出错。

解决方法： 我们需要定义一个新的 time.Time 类型，自定义的日期格式解析行为，包括 c.ShouldBindJSON / c.JSON / gorm写入Value / gorm写出Scan

gorm 内部是通过 Value 和 Scan 这两个方法完成值的写入和检出。那么从这个角度出发，我们就需要给我们的类型实现 Value 和 Scan 方法，分别对应写入的时候获取值和检出的时候解析值。

gorm中对应的时间类型也要使用MyLocalTime，才能正常使用自定义的 Value和 Scan 方法

这时候你的 go 应用已经可以正常存取自定义的日期格式格式了，但是如果传入一个空字符串 "" 日期数据，也会通过校验，并在数据库写入 null！这个问题是因为 gin 内置的 validator 对我们的 model.LocalTime 还没有一个完善的空值检测机制，我们只需要加上这个检测机制即可。（实现如下）

自定义中间件

如果需要自定义engine，不使用默认自带的中间件gin.Logger()和gin.Recovery()等，则可以使用gin.New()创建空干净的engine，自定义logger或recovery动作，可以参考下这篇文章 自定义错误处理

自定义中间件，需要函数返回gin.HandlerFunc类型，在c.Next() 前后位置分别自定义request前后的逻辑。

在中间件中如果不想继续后续接口的调用不能直接return，而是应该调用c.Abort()方法。如果需要返回状态码和对应的消息体，可以使用c.AbortWithStatusJSON()方法。

规划项目结构

这面的目录结构是我自定义的，大家也可以根据自己的习惯去定义。推荐大家看看这篇文章领域驱动设计在互联网业务开发中的实践

  • config - 定义项目的一些参数配置
  • entity - 对提交过来的数据进行验证，然后将验证完成的数据传递给 service 处理
  • middleware - 存放自定义的中间件
  • model - 数据库ORM
  • router - 项目API路由
  • service - 业务层：定制化一些业务逻辑
  • utils - 自定义一些工具包
