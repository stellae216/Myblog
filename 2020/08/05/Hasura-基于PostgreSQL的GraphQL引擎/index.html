Hasura-基于PostgreSQL的GraphQL引擎

本文最后更新于：4 年前

Hasura简介

Hasura GraphQL引擎是一种快速的GraphQL服务器，通过Postgres为您提供即时，实时的GraphQL API，并具有针对数据库事件的webhook触发器和业务逻辑的远程模式。

Hasura可帮助您构建由Postgres支持的GraphQL应用程序，或使用Postgres将其逐步移至GraphQL以用于现有应用程序。

如果你还不知道什么是GraphQL，可以去看下我的另外一片文章《为什么要使用GraphQL》

Hasura特性

Hasura具有许多功能，可帮助您快速采用GraphQL和3factor架构：

  • 进行强大的查询：内置过滤，分页，模式搜索，批量插入，更新，删除突变
  • 实时：使用订阅将任何GraphQL查询转换为实时查询
  • 合并远程模式：通过单个GraphQL Engine端点访问业务逻辑的自定义GraphQL模式。阅读更多。
  • 触发webhooks或无服务器功能：在Postgres上插入/更新/删除事件（了解更多）
  • 与现有的实时数据库兼容：将其指向现有的Postgres数据库，以立即获得可立即使用的GraphQL API
  • 细粒度的访问控制：与您的auth系统集成的动态访问控制（例如：auth0，firebase-auth）
  • 高性能和低占用空间：〜15MB docker映像；〜50MB RAM @ 1000请求/秒; 多核意识
  • 管理员界面和迁移：管理员界面和受Rails启发的架构迁移
  • Postgres❤️：支持Postgres类型（PostGIS /地理定位等），将视图转换为图形，触发具有突变的存储函数或过程

Hasura常用命令

  • 打开hasura控制台
  • 查看migrate状态
  • 升级migrate
  • 更新metadata
  • 创建migrate
  • 回退migrate版本

远程操作：<hasura command> --endpoint <url> --admin-secret <key>
