Git基础命令

本文最后更新于：1 年前

Git是一个版本控制工具 – 分布式的版本控制系统

Mercury – python开发的版本控制软件

配置邮箱和用户名
git config –global user.name
git config –global user.email

Git常用操作

  • git init – 初始化 – 将一个文件夹初始化成git本地仓库
  • git add - 将文件添加到暂存区
  • git reset HEAD - 将文件从暂存区移除
  • git checkout – - 将暂存区的文件恢复到工作区
  • git status - 查看暂存区的状态
  • git commit - 将暂存区的内容提交到本地仓库
  • git branch - 查看当前分支
  • git checkout -b 创建分支并切换
  • git log - 查看提交日志（当前版本之前的版本）
  • git reflog - 查看日志（所有的版本）
  • git reset - 回到指定版本
  • git reset –soft HEAD～1 - 撤销commit，不会修改代码，数量代表撤销次数
  • git HEAD^ - 回到上一个版本
  • git clone - 克隆
  • git push - 推 / 上传
  • git pull - 拉 / 下载
  • git fetch - 取
  • git rebase - 变基

一般公司里面会自己用gitlab搭建自己的git私服

创建git项目后，需要对项目进行初始版本更新，比如生成 .gitignore文件，它可以忽略一些不被纳入版本控制的文件，一般我们可以在 https://www.gitignore.io/网站中生成专业的忽略文件信息，输入对应的语言和工具即可。并且把master设置为保护状态。

在开发过程中如果要同步master分支上他人提交的工作成果，可以使用以下两种方式：

  1. git pull origin master
  2. git fetch / git rebase master
    如果有冲突，可以选择接受/保留/手动合并
    git rebase –continue 继续变基
    git rebase –abort 放弃变基

Git的规范流程(分支管理策略)

如果不清楚，可以阅读阮一峰老师的Git 工作流程

注意：绝对不允许在master分支上直接做开发

  • Git-flow

两个长线分支：master / develop

三个短线分支： hotfix / feature / release

  • Github-flow

一般我们采用Github-flow规范流程

  1. 克隆项目到本地并基于master分支创建自己的工作分支。
    • git clone <仓库的URL>
    • git clone -o <远端服务器名> <仓库的URL>
    • git branch -a 查看所有分支
    • git branch <分支名> 创建 / git checkout -b <分支名> 创建并切换
  2. 在自己的分支上做开发并实施本地版本控制。
    • git add .
    • git status 查看工作区和暂存区的区别
    • git commit -m ‘’
    • git log –graph –abbrev-commit
  3. 将自己的分支推送到服务器。
    • git push origin <分支名>:<服务器上的分支名>
  4. 发起一次合并请求，合并工作成果到master分支(线上完成)
  5. 项目管理者和测试人员会分别审查和测试代码，然后决定是否接受合并请求。
